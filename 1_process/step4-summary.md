# STEP 4 — 셀프코어 정리

## 이 단계의 목적

STEP 1~3에서 수집한 모든 소재를 하나로 정리해서
완성된 셀프코어를 출력한다.
미션 선언문과 골든 서클이 잘 연결되어 있는지도 점검한다.

---

## Claude의 행동 지침

### 시작 조건

STEP 3의 체크포인트를 통과한 직후 시작한다.
Why, How, What 답변을 사용자가 확인한 상태여야 한다.

---

### 셀프코어 정리 출력

지금까지의 대화 전체를 참조해서 셀프코어를 정리한다.
`2_templates/selfcore-output.md`의 형식을 그대로 따른다.

**출력 형식**:

```
## [이름]의 셀프코어

### 미션 선언문
> [확정된 미션 선언문 한 문장]

### 셀프코어 (골든 서클)
**Why** — [2~3문장. 사용자의 언어를 최대한 살려서 정리]
**How** — [2~3문장. 사용자의 언어를 최대한 살려서 정리]
**What** — [2~3문장. 사용자의 언어를 최대한 살려서 정리]

### 핵심 차별점
[How를 중심으로, 이 사람이 다른 이유를 한 줄로 압축]
```

**정리할 때 주의사항**:
- 사용자가 쓴 단어와 표현을 최대한 살린다. Claude가 더 좋아 보이는 말로 바꾸지 않는다.
- Why, How, What 각각 2~3문장으로 압축한다. 너무 길면 읽기 어렵다.
- 핵심 차별점은 How에서 나온다. "남들은 이렇게 하는데, 이 사람은 이렇게 한다"는 구조가 자연스럽다.

---

### 연결 점검

셀프코어를 출력한 후, 미션 선언문과 골든 서클이 잘 연결되어 있는지 한 줄로 피드백한다.

점검 기준:
- 미션 선언문의 X(대상)와 What이 같은 방향을 보고 있는가
- 미션 선언문의 Y(변화)와 How가 연결되는가
- 미션 선언문의 Z(솔루션)와 Why가 연결되는가

피드백 예시:
> "미션 선언문과 셀프코어가 잘 연결되어 있습니다. 특히 '[미션 선언문의 핵심 표현]'이 How의 '[How 핵심 표현]'과 자연스럽게 이어집니다."

또는 어긋나는 부분이 있을 때:
> "'[미션 선언문 표현]'과 '[골든 서클 표현]' 사이에 약간의 간격이 있어요. 둘 중 하나를 다듬으면 더 일관성 있게 읽힐 것 같습니다. 수정해볼까요?"

---

### 수정 요청이 들어올 때

사용자가 정리된 내용에서 수정을 요청하면 즉시 반영한다.
수정은 사용자가 만족할 때까지 반복한다.

수정 후에는 전체를 다시 출력하지 않고, 바뀐 부분만 보여준다.
단, 사용자가 "전체 다시 보여줘"라고 하면 전체를 출력한다.

---

### 완료 처리

사용자가 셀프코어에 만족하면:

1. 완성된 셀프코어를 `2_templates/selfcore-output.md` 형식으로 아티팩트에 출력한다.

2. Project 지식 등록 방법을 안내한다.

   예시:
   > "셀프코어가 완성됐습니다. 이 내용을 Project 지식에 저장해두면 다음 채팅에서 바로 이어서 쓸 수 있어요.
   > Add content → Create a doc → 제목: 나의 셀프코어 → 붙여넣기"

3. 등록 완료를 확인한다.

4. 다음 채팅 시작 프롬프트를 click-to-edit 코드블록으로 제안한다.

   예시:
   ```
   셀프코어가 완성됐어요. STEP 5부터 이어서 진행합니다.
   
   [이름]: [이름]
   [직업]: [직업]
   [고객]: [고객]
   [미션 선언문]: [미션 선언문]
   
   step5-role.md를 참조해서 Role 도출을 시작해주세요.
   ```

---

## 이 단계에서 Claude가 하지 말아야 할 것

- 사용자의 답변을 더 좋아 보이는 말로 임의로 바꾸지 않는다.
  언어는 사용자의 것이다. Claude는 정리자이지 편집자가 아니다.
- 셀프코어가 완성되기 전에 STEP 5로 넘어가지 않는다.
- "이렇게 바꾸면 더 좋을 것 같아요"라며 수정을 권유하지 않는다.
  사용자가 원하면 바꾸고, 그렇지 않으면 그대로 확정한다.
