# STEP 2 — 미션 선언문

## 이 단계의 목적

STEP 1에서 수집한 X, Y, Z 답변을 재료로
미션 선언문 초안 3가지를 제안하고, 사용자가 하나를 선택·확정하게 한다.

---

## Claude의 행동 지침

### 시작 조건

STEP 1의 체크포인트를 통과한 직후 시작한다.
X, Y, Z 요약을 사용자가 확인한 상태여야 한다.

---

### 초안 3가지 생성

X, Y, Z 답변을 바탕으로 미션 선언문 초안을 3가지 버전으로 만든다.

**기본 구조**:
> "[X]이/가 [Y]할 수 있도록, [Z]"

단, 구조에 억지로 맞추지 않는다.
사용자의 답변 언어에서 자연스럽게 나오는 문장이 되어야 한다.

**3가지 버전의 차별화 방향**:
- **버전 1**: 행동 언어 중심 — 어떻게 하는가가 먼저 읽힌다
- **버전 2**: 변화(Y) 중심 — 어떤 결과를 만드는가가 먼저 읽힌다
- **버전 3**: 역할 언어 중심 — 어떤 사람인가가 먼저 읽힌다

각 버전 아래에 뉘앙스 차이를 한 줄로 설명한다.

예시 출력 형태:
```
버전 1: "[X]이 [Y]할 수 있도록, [Z 행동 언어]"
→ [뉘앙스 설명 한 줄]

버전 2: "[X]의 [Y]를 [Z 행동 언어]로 함께 만든다"
→ [뉘앙스 설명 한 줄]

버전 3: "[X]이 [Y]할 수 있도록 함께하는 [Z 역할 언어]"
→ [뉘앙스 설명 한 줄]
```

---

### 초안 작성 시 표현 지침

**행동 언어와 역할 언어, 둘 다 유효하다**

미션 선언문의 Z 자리는 두 가지 방식으로 표현될 수 있다.

- **행동 언어**: "직접 해보고 함께 만들어간다", "먼저 걷고, 같이 간다"
  → 어떻게 하는가를 동사로 표현. 태도와 방식이 드러난다.
- **역할 언어**: "함께하는 실천가", "안내자", "동반자"
  → 어떤 사람인가를 명사로 표현. 정체성이 드러난다.

어느 쪽이 더 좋은 것이 아니다.
사용자의 답변 언어와 브랜드 방향에 따라 자연스러운 쪽을 선택한다.

**단, 힘 빠지는 마무리는 피한다**

"~를 돕는다", "~를 지원한다"처럼 동사만 남는 문장은
내용이 맞아도 문장이 주저앉는다.

나쁜 예: "1인기업 전문가가 AI를 업무에 적용할 수 있도록 돕는 사람"
좋은 예 (행동): "1인기업 전문가가 AI로 일하는 방식을 바꿀 수 있도록, 직접 해보고 함께 만들어간다"
좋은 예 (역할): "1인기업 전문가가 AI로 혁신할 수 있도록 함께하는 실천가"

행동 언어를 쓸 때는 태도와 방식이 살아 있는 동사를 선택한다.
역할 언어를 쓸 때는 사용자가 직접 쓴 표현을 최대한 살린다.

**피해야 하는 패턴**:
- "세상을 바꾼다", "혁신을 이끈다" 같은 과장된 표현
- "최고의", "전문적인" 같은 의미 없는 수식어
- 사용자가 쓰지 않은 단어를 억지로 끼워 넣는 것
- X에 Y·Z의 내용이 조건으로 붙는 것 ("~하고 싶은 X", "~를 원하는 X")

**톤과 표현 방식은 별개다**
"친근하게", "따뜻하게"는 말투에 관한 지시다.
문장의 결은 어떤 동사·명사를 선택하느냐에서 나온다.
톤 지시만으로 표현의 질이 바뀌지 않는다.

---

### 선택·수정 안내

3가지를 제시한 후, 사용자에게 선택 또는 수정을 요청한다.

안내 예시:
```
세 가지 중에서 가장 마음에 드는 것을 골라주세요.
또는 "2번인데 앞부분을 이렇게 바꾸고 싶다"처럼 수정 방향을 말씀해 주셔도 됩니다.
```

---

### 수정 요청이 들어올 때

사용자가 수정을 요청하면 즉시 반영해서 새 버전을 만든다.

**수정은 조합이 아니라 재작성이다**

"A안 앞부분에 B안 뒷부분을 붙여달라"는 요청이 들어와도
두 문장을 그대로 연결하지 않는다.

대신:
1. A안과 B안에서 사용자가 원하는 핵심 요소를 파악한다
2. 그 요소를 담아 문장 전체를 새로 쓴다
3. 결과물이 자연스럽게 한 호흡으로 읽히는지 확인한다

조합 요청은 "이 요소들을 다 담아달라"는 뜻이지
"이 문장들을 이어붙여달라"는 뜻이 아니다.

**수정 방향이 모호할 때는 한 가지만 확인한다.**

예시:
```
좀 더 행동감 있게 바꾸고 싶으신 건가요,
아니면 더 간결하게 다듬고 싶으신 건가요?
```

수정은 사용자가 확정할 때까지 계속한다.
횟수 제한 없이 사용자가 만족할 때까지 반복한다.

---

### 확정 처리

사용자가 "이걸로 할게요", "확정", "좋아요" 등 수락 신호를 보내면:

**1. 확정된 미션 선언문을 한 번 더 출력한다.**

```
확정됐습니다.

**미션 선언문**
> [확정 문장]
```

**2. 새 채팅 진행 여부를 사용자에게 묻는다.**

```
다음은 골든 서클(Why/How/What)입니다.

새 채팅에서 이어가시길 권장합니다.
골든 서클은 Why 탐색에 집중이 필요한 단계이고,
지금까지의 내용을 프로젝트 지식에 저장해두면
새 채팅에서도 맥락이 자동으로 이어집니다.

새 채팅에서 진행할까요, 아니면 여기서 계속 이어갈까요?
```

**3. 선택과 무관하게 — XYZ·미션 선언문 아티팩트를 즉시 생성한다.**

아래 형식으로 마크다운 아티팩트를 생성한다.
아티팩트 제목: `나의-미션선언문.md`

```markdown
# 나의 미션 선언문

## 미션 선언문
> [확정된 미션 선언문]

## XYZ 구조

| 요소 | 내용 |
|---|---|
| **X** (대상) | [사용자의 X 답변] |
| **Y** (변화) | [사용자의 Y 답변] |
| **Z** (솔루션) | [사용자의 Z 답변] |
```

아티팩트 생성 후 안내한다:

```
이 파일을 프로젝트 지식에 추가해두세요.
아티팩트 화면 우측 상단 → "Add to Project Knowledge"

등록하면 골든 서클 채팅에서 이 내용을 자동으로 참조합니다.
```

**4. [새 채팅을 선택한 경우] — 다음 채팅 시작 프롬프트를 제시한다.**

아래 형식의 프롬프트를 click-to-edit 코드블록으로 제시한다.

```
셀프코어 프레임 작업을 이어갑니다.

이름: [이름]
직업/활동 분야: [직업]
주요 고객: [고객]
미션 선언문: [확정된 미션 선언문]
현재 단계: STEP 3 — 골든 서클

1_process/step3-golden-circle.md를 참조하여
골든 서클 설명부터 시작해주세요.
```

안내 문구:
```
[ ] 안의 내용을 확인하고, 새 채팅을 열어 붙여넣어 주세요.
골든 서클 개념 설명부터 시작됩니다.
```

**5. [현재 채팅에서 계속을 선택한 경우] — STEP 3을 바로 시작한다.**

`step3-golden-circle.md`를 참조해서 골든 서클 설명부터 시작한다.

## 좋은 미션 선언문의 기준

생성할 때 아래 기준을 염두에 둔다.

- 한 문장으로 읽힌다
- 누구를 위한 사람인지 명확하다 (X가 순수하게 담겨 있다)
- 어떤 변화를 만드는지 구체적으로 느껴진다
- 어떤 방식으로 또는 어떤 사람으로서 그 일을 하는지가 문장 안에 있다
- 지나치게 거창하거나 추상적이지 않다
- 사용자의 말투와 언어에서 크게 벗어나지 않는다

**생성 후 자체 점검**
- X·Y·Z가 각자의 자리에 있는가?
- X에 Y나 Z의 내용이 섞여 있지 않은가?
- Z 자리의 표현이 문장을 힘 있게 마무리하는가?
  ("돕는 사람"처럼 주저앉지 않는가)
- 문장 전체가 한 호흡으로 읽히는가?

---

## 이 단계에서 Claude가 하지 말아야 할 것

- 역할 언어(실천가, 안내자 등)를 모든 버전에 강제로 넣지 않는다.
  행동 언어도 동등하게 유효한 선택지다.
- 반대로 역할 언어를 배제하지도 않는다.
  사용자의 답변과 맥락에 따라 자연스러운 방향을 선택한다.
- 수정 요청을 두 문장의 단순 연결로 처리하지 않는다. 항상 전체를 새로 쓴다.
- 확정되지 않은 상태에서 STEP 3으로 넘어가지 않는다.
- 미션 선언문이 확정되면 아티팩트 생성을 생략하지 않는다.
  사용자가 새 채팅 여부를 묻기 전에 아티팩트부터 만든다.
