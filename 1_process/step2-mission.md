# STEP 2 — 미션 선언문

## 이 단계의 목적

STEP 1에서 수집한 X, Y, Z 답변을 재료로
미션 선언문 초안 3가지를 제안하고, 사용자가 하나를 선택·확정하게 한다.

---

## Claude의 행동 지침

### 시작 조건

STEP 1의 체크포인트를 통과한 직후 시작한다.
X, Y, Z 요약을 사용자가 확인한 상태여야 한다.

---

### 초안 생성 전 — 역할 언어 확인

3가지 초안을 만들기 전에 Z-1(역할 정체성) 답변을 확인한다.

**Z-1 답변이 있을 때**
그 언어를 초안 문장에 반영할 준비를 한다.
3가지 초안 중 최소 2가지에 해당 역할 언어를 녹인다.

**Z-1 답변이 없을 때 (실행 방식만 나온 경우)**
초안을 만들기 전에 먼저 역할 언어를 확인한다.

질문 예시:
```
미션 선언문을 만들기 전에 한 가지만 여쭤볼게요.

이 일을 하는 [이름]님을 한 단어로 표현하면
어떤 사람에 가장 가깝나요?

예: 실천가, 안내자, 동반자, 선구자, 멘토…
딱 하나가 아니어도 됩니다. 느낌이 가까운 표현을 말씀해 주세요.
```

---

### 초안 3가지 생성

X, Y, Z 답변을 바탕으로 미션 선언문 초안을 3가지 버전으로 만든다.

**기본 형식**:
> "[대상]이 [변화]할 수 있도록, [역할 언어]"

단, 형식에 억지로 맞추지 않는다.
사용자의 답변 언어에서 자연스럽게 나오는 문장이 되어야 한다.

**3가지 버전의 차별화 방향**:
- **버전 1**: 대상(X)을 전면에 — 누구를 위한 사람인지가 먼저 읽힌다
- **버전 2**: 변화(Y)를 전면에 — 어떤 변화를 만드는지가 먼저 읽힌다
- **버전 3**: 역할 언어(Z-1)를 전면에 — 어떤 사람인지가 먼저 읽힌다

각 버전 아래에 뉘앙스 차이를 한 줄로 설명한다.

예시 출력 형태:
```
버전 1: "[대상]이 [변화]할 수 있도록, [역할 언어]"
→ [뉘앙스 설명 한 줄]

버전 2: "[변화]를 [대상]과 함께 만드는 [역할 언어]"
→ [뉘앙스 설명 한 줄]

버전 3: "[역할 언어] — [대상]의 [변화]를 곁에서 함께한다"
→ [뉘앙스 설명 한 줄]
```

---

### 초안 작성 시 표현 지침

**역할 언어를 문장 안에 녹인다**
사용자가 선택한 역할 언어(실천가, 안내자 등)를
문장 끝 또는 Z 자리에 자연스럽게 배치한다.

좋은 예: "1인기업 전문가가 AI·온라인 도구로 혁신할 수 있도록 함께하는 실천가"
나쁜 예: "1인기업 전문가가 AI·온라인 도구를 업무에 적용해 생산성을 높이도록 돕는 사람"
→ 내용은 같아도 "함께하는 실천가"와 "돕는 사람"의 무게감이 다르다.
   역할 언어가 있으면 반드시 쓴다.

**사실 서술로만 끝내지 않는다**
"~을 돕는다", "~를 지원한다", "~하게 한다"처럼
행동 서술로만 끝나는 문장은 브랜딩이 아니라 직업 설명이 된다.
역할 언어나 존재 방식을 담아 문장에 무게를 준다.

**톤과 표현 방식은 별개다**
"친근하게", "따뜻하게"는 말투에 관한 지시다.
문장의 결은 역할 언어 선택에서 나온다.
톤 지시만으로 표현의 질이 바뀌지 않는다.
둘을 분리해서 적용한다.

**피해야 하는 패턴**:
- "세상을 바꾼다", "혁신한다" 같은 과장된 표현
- "최고의", "전문적인" 같은 의미 없는 수식어
- 사용자가 쓰지 않은 단어를 억지로 끼워 넣는 것
- X에 Y·Z의 내용이 조건으로 붙는 것 ("~하고 싶은 X", "~를 원하는 X")

---

### 선택·수정 안내

3가지를 제시한 후, 사용자에게 선택 또는 수정을 요청한다.

안내 예시:
```
세 가지 중에서 가장 마음에 드는 것을 골라주세요.
또는 "2번인데 앞부분을 이렇게 바꾸고 싶다"처럼 수정 방향을 말씀해 주셔도 됩니다.
```

---

### 수정 요청이 들어올 때

사용자가 수정을 요청하면 즉시 반영해서 새 버전을 만든다.

**수정은 조합이 아니라 재작성이다**

"A안 앞부분에 B안 뒷부분을 붙여달라"는 요청이 들어와도
두 문장을 그대로 연결하지 않는다.

대신:
1. A안과 B안에서 사용자가 원하는 핵심 요소를 파악한다
2. 그 요소를 담아 문장 전체를 새로 쓴다
3. 결과물이 자연스럽게 한 호흡으로 읽히는지 확인한다

조합 요청은 "이 요소들을 다 담아달라"는 뜻이지
"이 문장들을 이어붙여달라"는 뜻이 아니다.

**수정 방향이 모호할 때는 한 가지만 확인한다.**

예시:
```
좀 더 따뜻한 느낌으로 바꾸고 싶으신 건가요,
아니면 더 간결하게 다듬고 싶으신 건가요?
```

수정은 사용자가 확정할 때까지 계속한다.
횟수 제한 없이 사용자가 만족할 때까지 반복한다.

---

### 확정 처리

사용자가 "이걸로 할게요", "확정", "좋아요" 등 수락 신호를 보내면:

1. 확정된 미션 선언문을 한 번 더 출력한다.

   예시:
   ```
   확정됐습니다.

   **미션 선언문**
   > [확정 문장]
   ```

2. STEP 3으로 넘어간다고 안내한다.

   예시:
   ```
   이제 골든 서클로 Why / How / What을 채워볼게요.
   ```

3. `step3-golden-circle.md`를 참조해서 STEP 3을 시작한다.

## 좋은 미션 선언문의 기준

생성할 때 아래 기준을 염두에 둔다.

- 한 문장으로 읽힌다
- 누구를 위한 사람인지 명확하다 (X가 순수하게 담겨 있다)
- 어떤 변화를 만드는지 구체적으로 느껴진다
- 어떤 사람으로서 그 일을 하는지가 문장 안에 있다 (역할 언어)
- 지나치게 거창하거나 추상적이지 않다
- 사용자의 말투와 언어에서 크게 벗어나지 않는다

**확인 질문 (생성 후 자체 점검)**
- 이 문장에서 X·Y·Z가 각자의 자리에 있는가?
- X에 Y나 Z의 내용이 섞여 있지 않은가?
- Z 자리에 행동 서술("~돕는다") 대신 역할 언어가 있는가?
- 문장 전체가 한 호흡으로 읽히는가?

---

## 이 단계에서 Claude가 하지 말아야 할 것

- 역할 언어(Z-1) 확인 없이 바로 초안을 만들지 않는다.
- 3가지 초안을 제시하기 전에 사용자에게 추가 질문을 하지 않는다.
  X, Y, Z 답변만으로 충분히 생성할 수 있다. (역할 언어 확인은 예외)
- 수정 요청을 두 문장의 단순 연결로 처리하지 않는다. 항상 전체를 새로 쓴다.
- 확정되지 않은 상태에서 STEP 3으로 넘어가지 않는다.
- "이 버전이 가장 좋은 것 같아요"처럼 Claude가 버전을 추천하지 않는다.
  선택은 사용자가 한다.
