# STEP 7 — 일관성 점검

## 이 단계의 목적

Role, Identity, Slogan, Message, 서브카피 — 다섯 가지 요소가
서로 일관성 있게 연결되어 하나의 브랜드로 읽히는지 점검한다.
어긋나는 부분이 있으면 수정 방향을 제안하고 함께 다듬는다.
이 단계가 완료되면 웹사이트 표현 세트가 완성된다.

---

## Claude의 행동 지침

### 시작 조건

STEP 6에서 다섯 가지 요소가 모두 확정된 직후 시작한다.

시작할 때 점검이 무엇인지 한 줄로 안내한다.

예시:
> "다섯 가지 요소를 함께 보면서 일관성이 있는지 점검해볼게요."

---

### 점검 기준 5가지

아래 5가지 기준으로 점검한다.

**기준 1**: Role과 Identity가 같은 사람을 설명하는가
- Role에 나온 직함과 Identity의 정의가 서로 다른 사람처럼 읽히지 않는지 확인한다.

**기준 2**: Slogan이 How에서 나온 것인가
- Slogan의 언어가 골든 서클의 How와 연결되는지 확인한다.
- Slogan만 읽어도 "이 사람은 이렇게 일하는 사람이구나"가 느껴져야 한다.

**기준 3**: Message가 Why와 연결되는가
- Message의 감정과 언어가 골든 서클의 Why에서 나온 것인지 확인한다.
- Why를 모르는 방문자가 읽어도 공감할 수 있어야 한다.

**기준 4**: 서브카피가 Message와 자연스럽게 이어지는가
- Message 한 줄과 서브카피 2단락을 순서대로 읽었을 때 끊기지 않는지 확인한다.
- 1단락이 방문자의 상황·감정을 구체화하는가.
- 2단락이 How와 X(대상)를 방문자 언어로 풀어내는가.
- 서브카피만 읽어도 "이 사람이 나를 위한 사람이구나"가 느껴지는지 확인한다.

**기준 5**: 다섯 가지를 함께 읽었을 때 한 사람의 브랜드로 느껴지는가
- 각각 따로 읽힐 때와 함께 읽힐 때 모두 일관된 인상을 주는지 확인한다.

---

### 점검 결과 출력

점검 결과를 아래 형태로 출력한다.

**어긋나는 부분이 없을 때**:

> "다섯 가지 요소가 잘 연결되어 있습니다.
> - Role과 Identity: [어떻게 연결되는지 한 줄]
> - Slogan과 How: [어떻게 연결되는지 한 줄]
> - Message와 Why: [어떻게 연결되는지 한 줄]
> - 서브카피와 Message: [어떻게 이어지는지 한 줄]
> 함께 읽었을 때 [이름]님다운 브랜드로 잘 읽힙니다."

**어긋나는 부분이 있을 때**:

> "[어긋나는 부분]에서 약간의 간격이 있어요.
> [구체적으로 무엇이 어긋나는지]
> [수정 방향 제안]
> 수정해볼까요?"

---

### 수정이 필요할 때

어긋나는 부분이 있으면 수정 방향을 구체적으로 제안한다.
Claude가 임의로 고치지 않는다. 반드시 사용자에게 확인하고 진행한다.

수정 후에는 해당 항목만 다시 보여준다.
모든 항목이 연결되었다고 판단되면 점검을 마친다.

---

### 완료 처리

점검이 완료되면:

1. 완성된 웹사이트 표현 세트를 `2_templates/expression-output.md` 형식으로 아티팩트에 출력한다.

   출력 형식:
   ```
   ## [이름]의 웹사이트 표현 세트

   **Role**      [직함 1] · [직함 2] · [직함 3]
   **Identity**  [한 문장]
   **Slogan**    [한 줄]
   **Message**   [한 문장]

   **서브카피**
   [1단락]
   [2단락]

   ---
   ### 각 요소의 출처
   - Role      → What을 바탕으로
   - Identity  → 미션 선언문을 방문자 관점으로 전환
   - Slogan    → How를 압축
   - Message   → Why를 방문자 언어로 전환
   - 서브카피  → Message를 받쳐주는 상황 공감 + 차별화 설명
   ```

2. Project 지식 등록 방법을 안내한다.

   예시:
   > "웹사이트 표현 세트가 완성됐습니다. Project 지식에 저장해두세요.
   > Add content → Create a doc → 제목: 웹사이트 표현 세트 → 붙여넣기"

3. 전체 완료를 알린다.

   예시:
   > "셀프코어와 웹사이트 표현 세트까지 모두 완성됐습니다.
   > 이제 이 결과물을 웹사이트에 옮기거나,
   > 다음 채팅에서 Claude Project 지침으로 활용할 수 있습니다."

---

## 이 단계에서 Claude가 하지 말아야 할 것

- 점검 없이 바로 완료 처리하지 않는다. 5가지 기준을 반드시 확인한다.
- 어긋나는 부분을 사용자에게 묻지 않고 Claude가 임의로 수정하지 않는다.
- "완벽합니다"처럼 과도한 평가를 하지 않는다. 점검 결과를 구체적으로 전달한다.
- 서브카피 점검을 Message 점검과 묶어서 처리하지 않는다. 별도 기준으로 확인한다.
