# Brizy Story 활용 방안

> 템플릿 상세 분석은 `brizy-story-templates.md` 참조  
> 이 문서는 **언제, 왜, 어떻게 쓸 것인가**에 집중한다

---

## Brizy Story란 무엇인가

모바일에서 스와이프로 넘기는 미니 웹사이트다.  
인스타그램·유튜브 쇼츠처럼 세로형 슬라이드를 넘기는 방식으로, 링크 하나로 공유할 수 있다.  
앱 설치 없이 브라우저에서 바로 열린다.

---

## 핵심 전제 — 누구에게 어떤 도구인가

Brizy Story는 모든 사람에게 같은 가치를 주지 않는다.

### 도구 숙련자에게는 보조 수단

Jekyll, WordPress, Gamma, 뉴스레터 등 이미 여러 도구를 능숙하게 쓰는 사람에게는  
**이미 만든 것들의 모바일 친화적 입구** 역할로 의미가 있다.

- 강의 커리큘럼을 한눈에 보여주는 요약본
- 웨비나 진행 순서 안내
- 웹사이트·포트폴리오의 모바일 명함

### 도구 비숙련자에게는 게임 체인저

네이버 블로그가 전부였던 강사에게 Brizy Story는  
**처음으로 "내 것"이 생기는 경험**이다.

| 기존 방식 | Brizy Story로 |
|---|---|
| 네이버 블로그 공지글 | 서비스 소개 스토리 |
| 카카오톡 긴 문자 소개 | 모바일 명함 링크 |
| 없음 | 랜딩페이지 |
| 없음 | 포트폴리오 |
| 없음 | 강의 모집 페이지 |

만들 수 있는 것이 아무것도 없었던 사람이 Brizy Story 하나로 모든 것을 할 수 있게 된다.  
이것이 이 도구의 진짜 가치다.

## 1인기업 강사·코치·컨설턴트의 실전 활용 4가지

### ① 모바일 명함

> **"링크 하나로 나를 소개한다"**

처음 만나는 사람에게 "이 링크 보내드릴게요" 한 마디로 자기소개가 끝난다.  
QR 없이 카카오톡으로 바로 공유 가능하다.

**스토리 흐름**
```
나는 누구인가 (이름·사진·한 줄 포지셔닝)
→ 내가 하는 일
→ 내가 도울 수 있는 것
→ 나를 믿어도 되는 이유 (후기·성과)
→ 연락처·SNS·링크
```

**이런 분들에게 특히 유효**
- 명함을 들고 다니기 어려운 온라인 네트워킹 환경
- 자기소개 자료를 카카오톡으로 공유하는 경우
- 여러 역할을 하는 1인기업 (강사이자 코치이자 작가)

---

### ② 교육 신청 랜딩페이지 + 커리큘럼 소개

> **"설명하지 말고, 보여줘라"**

별도 홈페이지 없이 단일 서비스 랜딩 역할을 한다.  
커리큘럼을 슬라이드 한 장씩 순차로 보여주는 방식이 How-to 안내에 특히 효과적이다.

**스토리 흐름**
```
프로그램 이름·슬로건
→ 이런 분들을 위한 프로그램입니다
→ 무엇을 얻게 되나요 (결과)
→ 커리큘럼 (회차별 or 단계별)
→ 강사 소개
→ 신청 CTA
```

**활용 포인트**
- 웨비나 모집 공고 → Night Jam, Creative 구조 참고
- 강의 커리큘럼 → Language, 5 Ways, Detox Plan 구조 참고
- 챌린지 프로그램 → 날짜별·단계별 구조로 구성

---

### ③ 서비스 소개

> **"내가 뭘 하는 사람인지 3분 안에 전달한다"**

긴 소개 페이지 대신 스와이프 몇 번으로 핵심을 전달한다.  
SNS 링크, 오픈채팅방 안내 등에 붙여서 공유할 수 있다.

**스토리 흐름**
```
내가 하는 일 (포지셔닝)
→ 구체적으로 도울 수 있는 것들
→ 이런 분들이 저를 찾아요
→ 성과·후기
→ 문의·연락처
```

---

### ④ 포트폴리오

> **"숫자와 결과로 신뢰를 만든다"**

고객 후기, 작업물, 수치 성과를 슬라이드로 시각화한다.  
제안서 대신 "이 링크 보내드릴게요"로 대체할 수 있다.

**스토리 흐름**
```
대표 성과 or 작업물
→ 사례 1 (Before·After or 수치)
→ 사례 2
→ 고객 후기 (실명·별점)
→ 연락처
```

## 보조 활용 — 콘텐츠 크리에이터 관점

### 뉴스레터·블로그 미리보기

새 콘텐츠를 스토리로 예고해서 클릭을 유도한다.  
"이번 주 뉴스레터에 이런 내용이 있어요" → 링크로 공유.

### 추천 큐레이션

"강사가 추천하는 5가지 AI 도구", "이번 달 읽은 책 3권"  
Blockbusters 구조처럼 번호형 큐레이션 스토리로 만들면 공유 가치가 생긴다.

### 행사·웨비나 사전 예고

일정·연사·참가비·신청 링크를 슬라이드로 순차 공개.  
카카오톡 단체방, 오픈채팅, SNS DM으로 공유하기에 최적이다.

---

## 공유 방법

| 방법 | 설명 |
|---|---|
| 카카오톡 링크 | URL 복사 → 메시지로 전송 |
| SNS 바이오 링크 | 인스타그램·링크드인 프로필에 URL 삽입 |
| QR 코드 | URL → QR 생성 → 명함·PPT에 삽입 |
| 뉴스레터 CTA | 뉴스레터 하단에 버튼으로 삽입 |
| 웹사이트 링크 | 블로그·웹사이트의 특정 페이지에서 연결 |

---

## Brizy Story가 적합하지 않은 경우

- 긴 텍스트 설명이 필요한 상세 페이지 (블로그·웹사이트로)
- 정기적으로 콘텐츠가 업데이트되는 아카이브 (뉴스레터·블로그로)
- SEO가 중요한 검색 유입 콘텐츠 (웹사이트 포스트로)
- 복잡한 결제·폼이 필요한 신청 페이지 (랜딩페이지 빌더로)

Brizy Story는 **단방향으로 임팩트 있게 전달**하는 데 강하다.  
깊은 정보 전달이나 검색 유입이 필요한 용도는 다른 도구가 낫다.

## 실습 순서 제안

처음 만드는 사람이라면 이 순서가 좋다.

```
1단계  모바일 명함 (가장 개인적이고, 바로 쓸 수 있다)
2단계  서비스 소개 (명함과 내용이 겹쳐 빠르게 만들 수 있다)
3단계  교육 랜딩 or 포트폴리오 (목적이 명확해야 만들기 쉽다)
```

모바일 명함 하나만 잘 만들어도 충분하다.  
나머지는 필요가 생길 때 만들면 된다.
