# Brizy Story — 교육 신청 랜딩페이지 + 커리큘럼 소개 제작 가이드

> selfcore-frame / 4_brizy / guide-edu-landing.md
> 교육 랜딩페이지 제작 시 Claude가 참조하는 절차 안내

---

## 이 파일의 역할

사용자가 교육 신청 랜딩페이지 또는 커리큘럼 소개 스토리 제작을 요청할 때
Claude가 따르는 3단계 절차를 정의한다.
템플릿은 고정하지 않는다. 교육의 성격과 목적에 맞게 매번 새로 선정한다.

---

## 전체 흐름

```
1단계  콘텐츠 설계   셀프코어 확인 → 교육 정보 협의 → 슬라이드 흐름 확정
         ↓
2단계  템플릿 선정   구성에 맞는 템플릿 분석 → 후보 제안 → 사용자 확정
         ↓
3단계  가이드 생성   확정된 템플릿 구조로 슬라이드별 텍스트 표 출력
```

---

## 1단계 — 콘텐츠 설계

### 셀프코어 확인

Project 지식에서 셀프코어와 웹사이트 표현 세트를 먼저 찾는다.

- **있으면** → 내용을 요약해서 보여주고 확인 후 다음으로 넘어간다
- **없으면** → 아래 핵심 질문으로 먼저 뽑는다

```
셀프코어 없을 때 핵심 질문

1. 이 교육은 어떤 분들을 위한 것인가요? (대상)
2. 수강 후 어떤 변화가 생기길 바라나요? (변화)
3. 다른 교육과 무엇이 다른가요? (차별점)
```

### 교육 정보 협의

셀프코어 확인 후 아래 항목을 하나씩 확인한다.

**① 이 스토리의 주된 역할**
```
신청을 바로 받는 랜딩페이지인가요,
아니면 커리큘럼을 설명하는 소개 페이지인가요?
(또는 두 가지를 함께 담고 싶은가요?)
```

**② 교육 기본 정보**
```
- 교육명
- 형태 (온라인 / 오프라인 / 줌)
- 회차 및 시간
- 참가비 (있다면)
- 신청 마감 또는 모집 기간
```

**③ 커리큘럼 구성**
```
회차별 주제나 단계별 내용이 있으면 알려주세요.
없으면 셀프코어 What을 바탕으로 함께 구성합니다.
```

**④ 강조하고 싶은 것**
```
가격 / 신청 마감 / 강사 전문성 / 수강 후 변화 / 후기
중에서 가장 앞세우고 싶은 것은 무엇인가요?
```

**⑤ 톤**
```
전문적·신뢰감 / 따뜻하고 친근 / 동기부여·에너지 / 혼합
```

### 슬라이드 흐름 확정

위 답변을 바탕으로 슬라이드 흐름 초안을 제안하고 사용자와 확정한다.

흐름 예시 (목적과 강조점에 따라 달라진다):

```
[신청 전환 중심 흐름]
커버 → 이런 분들을 위한 교육 → 수강 후 변화 → 커리큘럼 → 강사 소개 → 참가비·일정 → 얼리버드·마감 → CTA

[커리큘럼 소개 중심 흐름]
커버 → 왜 이 교육인가 (Why) → 회차별 커리큘럼 → 수강 방식 → 강사 소개 → CTA

[간결형 흐름 (5~6장)]
커버 → 교육 핵심 가치 → 커리큘럼 요약 → 강사 소개 → 신청 CTA
```

흐름이 확정되면 2단계로 넘어간다.

---

## 2단계 — 템플릿 선정

### 분석 기준

확정된 슬라이드 흐름과 톤을 기준으로 `brizy-story-templates.md`에서 적합한 템플릿을 분석한다.

| 기준 | 확인할 것 |
|---|---|
| 교육·모집 구조 | 가치 제안 → 커리큘럼 → CTA 흐름이 있는가 |
| CTA 위치 | 신청 유도 슬라이드가 마지막에 명확히 있는가 |
| 텍스트 비중 | 커리큘럼 설명이 충분히 들어갈 수 있는 구조인가 |
| 이미지 의존도 | 강의 화면·사진 없이도 완성될 수 있는가 |
| 톤 | 사용자가 원하는 분위기와 맞는가 |

### 참고 템플릿 풀

교육 랜딩에 적합한 것으로 분석된 템플릿들. 이 중에서 선정하되, 다른 템플릿도 필요하면 분석한다.

| 템플릿 | 특징 | 적합한 상황 |
|---|---|---|
| The Academy | 커버→환경→프로그램→수강경험→철학→CTA 완결 구조 | 정규 교육 과정, 아카데미형 |
| Yoga | 오픈공지→철학→가치제안→프로그램→방식→CTA | 모집 공지 중심, 간결하게 |
| Creative | 인물→게스트/연사→행사상세→얼리버드→슬로건→일정 | 강의 오픈, 네트워킹 행사 |
| Night Jam | 커버→라인업→참가비→파트너→CTA (5장) | 웨비나, 단기 행사 |
| Language | 번호형 팁/단계 구조 (6장) | 커리큘럼 단계별 소개 |
| Detox Plan | 단계형 구조 (6장) | N주 챌린지, 단계별 프로그램 |
| Singapore | What/Where/How 카테고리 구조 | 프로그램 구성 설명 |
| Hiring | 직무/프로그램별 상세 안내 구조 | 모집형, 프로그램 상세 소개 |

### 1개 vs 2개 조합 결정

| 방식 | 언제 적합한가 |
|---|---|
| **1개 템플릿** | 하나의 템플릿으로 신청 → CTA 흐름이 충분히 커버될 때 |
| **2개 조합** | 강사 소개 파트와 커리큘럼 파트의 성격이 달라서 나눠야 할 때 |
| **브릿지 슬라이드** | 2개 조합일 때, 두 템플릿 사이 전환이 필요할 때 |

### 후보 제안 방식

후보 2~3개를 아래 형식으로 제시한다.

```
[후보 A] 템플릿명 (슬라이드 수)
구조: Slide 1 → Slide 2 → ...
이 조합을 추천하는 이유: ...
조정이 필요한 부분: ...

[후보 B] ...
```

사용자가 확정하면 3단계로 넘어간다.

---

## 3단계 — 슬라이드별 가이드 생성

확정된 템플릿 구조와 슬라이드 흐름을 기반으로
교육 정보와 셀프코어 내용을 채운 슬라이드별 표를 생성한다.

### 출력 형식

```
### Slide N — [슬라이드 제목] `[참조 템플릿 슬라이드]`

**역할:** [이 슬라이드가 랜딩페이지에서 하는 역할]

| 요소 | 내용 |
|---|---|
| [요소명] | [교육 정보 또는 셀프코어에서 가져온 내용] |
| [요소명] | [교육 정보 또는 셀프코어에서 가져온 내용] |

> **팁:** [이 슬라이드 수정 시 참고할 것]
```

### 커리큘럼 슬라이드 출력 시 주의사항

- 회차별 내용이 있으면 슬라이드 한 장에 한 회차씩 배치한다
- 회차가 많아서 장 수가 늘어나면 "요약형"과 "상세형" 두 가지 방식을 제안한다
- 요약형: 전체 커리큘럼을 한 장에 목록으로
- 상세형: 회차별로 한 장씩

### CTA 슬라이드 출력 시 주의사항

- 신청 링크, 마감일, 참가비를 반드시 포함한다
- 얼리버드가 있으면 별도 슬라이드로 분리하는 것이 효과적이다

---

## Claude가 하지 말아야 할 것

- 교육 기본 정보(대상·일정·참가비) 확인 없이 슬라이드를 제안하지 않는다
- 커리큘럼이 없는 상태에서 커리큘럼 슬라이드를 임의로 채우지 않는다
- The Academy나 Yoga를 기본값으로 고정해서 제안하지 않는다
- 슬라이드 흐름 확정 전에 템플릿을 먼저 선정하지 않는다
- 후보를 1개만 제안하지 않는다. 최소 2개 이상을 비교할 수 있게 제시한다
