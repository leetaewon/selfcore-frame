# Brizy Story — 모바일 명함 제작 가이드

> selfcore-frame / 4_brizy / guide-mobile-card.md
> 모바일 명함 제작 시 Claude가 참조하는 절차 안내

---

## 이 파일의 역할

사용자가 모바일 명함 제작을 요청할 때 Claude가 따르는 3단계 절차를 정의한다.
템플릿은 고정하지 않는다. 콘텐츠 구성과 사용자 성격에 맞게 매번 새로 선정한다.

---

## 전체 흐름

```
1단계  콘텐츠 설계   셀프코어 확인 → 구성 요소 협의 → 슬라이드 흐름 확정
         ↓
2단계  템플릿 선정   구성에 맞는 템플릿 분석 → 후보 제안 → 사용자 확정
         ↓
3단계  가이드 생성   확정된 템플릿 구조로 슬라이드별 텍스트 표 출력
```

---

## 1단계 — 콘텐츠 설계

### 셀프코어 확인

Project 지식에서 셀프코어(미션 선언문, Why/How/What, 웹사이트 표현 세트)를 먼저 찾는다.

- **있으면** → 내용을 요약해서 보여주고 확인 후 다음으로 넘어간다
- **없으면** → 아래 핵심 질문으로 먼저 뽑는다

```
셀프코어 없을 때 핵심 질문

1. 어떤 분들을 주로 돕고 있나요? (대상)
2. 그분들에게 어떤 변화가 생기길 바라나요? (변화)
3. 나만의 방식이 있다면 무엇인가요? (차별점)
4. 대표적인 성과나 작업물이 있다면? (신뢰 증거)
```

### 구성 요소 협의

셀프코어를 바탕으로 명함에 담을 내용을 함께 결정한다.
상세 텍스트를 작성하기 전에 어떤 슬라이드로 구성할지를 먼저 합의한다.

아래 항목을 하나씩 확인한다.

**① 명함의 주된 목적**
```
어떤 상황에서 주로 공유하실 건가요?
예) 처음 만난 사람에게 / SNS 바이오에 연결 / 강의 신청자에게 / 제안서 대신
```

**② 강조하고 싶은 것**
```
명함을 본 사람이 가장 먼저 느꼈으면 하는 것은 무엇인가요?
예) 이 사람이 누구인지 / 어떤 일을 하는지 / 왜 믿을 수 있는지 / 어떻게 연락하는지
```

**③ 슬라이드 수 선호**
```
간결하게 (5~6장) vs 충분히 설명하는 (8~10장)
```

**④ 톤**
```
전문적·단정 / 따뜻하고 친근 / 감성적 / 혼합
```

### 슬라이드 흐름 확정

위 답변을 바탕으로 슬라이드 흐름 초안을 제안하고 사용자와 확정한다.

흐름 예시 (목적·강조점에 따라 달라진다):

```
[자기소개 중심 흐름]
커버 → 나는 이런 사람 → 내가 도울 수 있는 것 → Why(브릿지) → 내 방식 → 성과/결과물 → 후기 → CTA

[신뢰 구축 중심 흐름]
커버 → 포지셔닝 → 구체적 서비스 → 성과/수치 → 고객 후기 → 내 방식 → CTA

[간결형 흐름]
커버 → 내가 하는 일 → 도울 수 있는 것 → 후기/성과 → CTA
```

흐름이 확정되면 2단계로 넘어간다.

---

## 2단계 — 템플릿 선정

### 분석 기준

확정된 슬라이드 흐름과 톤을 기준으로 `brizy-story-templates.md`에서 적합한 템플릿을 분석한다.

판단 기준:

| 기준 | 확인할 것 |
|---|---|
| 슬라이드 구조 | 확정된 흐름과 슬라이드 순서가 얼마나 가까운가 |
| 이미지 의존도 | 사진이 없거나 적어도 완성될 수 있는가 |
| 텍스트 비중 | 전달할 내용이 잘 담기는 구조인가 |
| 톤 | 사용자가 원하는 분위기와 맞는가 |
| 인물 중심 여부 | 자기소개형 명함이라면 인물 사진 슬라이드가 있는가 |

### 1개 vs 2개 조합 결정

| 방식 | 언제 적합한가 |
|---|---|
| **1개 템플릿** | 하나의 템플릿으로 흐름이 충분히 커버될 때. 슬라이드를 일부 삭제·추가로 조정 |
| **2개 조합** | 소개 파트와 스토리 파트의 성격이 달라서 한 템플릿으로 담기 어려울 때 |
| **브릿지 슬라이드** | 2개 조합일 때, 두 템플릿 사이의 톤·스타일 차이가 클 때 사이에 삽입 |

### 후보 제안 방식

후보 2~3개를 제안할 때 아래 형식으로 제시한다.

```
[후보 A] 템플릿명 (슬라이드 수)
구조: Slide 1 → Slide 2 → ...
이 조합을 추천하는 이유: ...
조정이 필요한 부분: ...

[후보 B] ...
```

사용자가 확정하면 3단계로 넘어간다.

---

## 3단계 — 슬라이드별 가이드 생성

확정된 템플릿 구조와 슬라이드 흐름을 기반으로
셀프코어 내용을 채운 슬라이드별 표를 생성한다.

### 출력 형식

각 슬라이드마다 아래 형식으로 출력한다.

```
### Slide N — [슬라이드 제목] `[참조 템플릿 슬라이드]`

**역할:** [이 슬라이드가 명함에서 하는 역할]

| 요소 | 내용 |
|---|---|
| [요소명] | [셀프코어에서 가져온 내용] |
| [요소명] | [셀프코어에서 가져온 내용] |

> **팁:** [이 슬라이드 수정 시 참고할 것]
```

### 브릿지 슬라이드 출력 형식 (해당하는 경우)

```
### Slide N — 브릿지 (직접 제작)

**역할:** 두 템플릿 사이의 전환. Why를 한 문장으로 전달한다.
**디자인:** 텍스트 중심, 심플하게. 배경색은 두 템플릿의 중간 톤.

| 요소 | 내용 |
|---|---|
| 큰 텍스트 | [Why에서 뽑은 한 문장] |
| 작은 텍스트 | [보조 문장] |

> **브릿지 유형:** 명언형 / 구분선형 / 질문형 중 선택
> - 명언형: 핵심 Why를 한 문장으로 — 감성적 전환이 필요할 때
> - 구분선형: "파트 2 — 나의 이야기" 같은 섹션 제목 — 구조를 명확히 보여줄 때
> - 질문형: "왜 이 일을 하는 걸까요?" — 스토리텔링으로 끌어들일 때
```

---

## Claude가 하지 말아야 할 것

- 셀프코어 확인 없이 템플릿부터 추천하지 않는다
- 콘텐츠 구성 협의 없이 Dating + Freelancer를 기본값으로 제안하지 않는다
- 슬라이드 흐름 확정 전에 템플릿을 먼저 선정하지 않는다
- 템플릿 확정 전에 슬라이드별 가이드를 출력하지 않는다
- 후보를 1개만 제안하지 않는다. 최소 2개 이상을 비교할 수 있게 제시한다
