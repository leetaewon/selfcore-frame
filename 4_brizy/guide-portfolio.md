# Brizy Story — 포트폴리오 제작 가이드

> selfcore-frame / 4_brizy / guide-portfolio.md
> 포트폴리오 스토리 제작 시 Claude가 참조하는 절차 안내

---

## 이 파일의 역할

사용자가 포트폴리오 스토리 제작을 요청할 때 Claude가 따르는 3단계 절차를 정의한다.
템플릿은 고정하지 않는다. 포트폴리오의 성격과 보여주고 싶은 것에 맞게 매번 새로 선정한다.

---

## 전체 흐름

```
1단계  콘텐츠 설계   셀프코어 확인 → 포트폴리오 정보 협의 → 슬라이드 흐름 확정
         ↓
2단계  템플릿 선정   구성에 맞는 템플릿 분석 → 후보 제안 → 사용자 확정
         ↓
3단계  가이드 생성   확정된 템플릿 구조로 슬라이드별 텍스트 표 출력
```

---

## 1단계 — 콘텐츠 설계

### 셀프코어 확인

Project 지식에서 셀프코어와 웹사이트 표현 세트를 먼저 찾는다.

- **있으면** → 내용을 요약해서 보여주고 확인 후 다음으로 넘어간다
- **없으면** → 아래 핵심 질문으로 먼저 뽑는다

```
셀프코어 없을 때 핵심 질문

1. 어떤 분야에서 어떤 결과물을 만들어왔나요? (전문 분야 + 결과물)
2. 이 포트폴리오를 보는 사람이 어떤 행동을 하길 바라나요? (목적)
3. 가장 자신 있게 보여줄 수 있는 것은 무엇인가요? (대표 성과)
```

### 포트폴리오 정보 협의

셀프코어 확인 후 아래 항목을 하나씩 확인한다.

**① 포트폴리오의 주된 목적**
```
이 포트폴리오를 누구에게, 어떤 상황에서 보여줄 건가요?
예) 새로운 클라이언트에게 / 강의 신청자에게 / SNS 팔로워에게 / 협업 제안 시
```

**② 보여줄 것의 성격**
```
완성된 작업물(이미지·결과물 중심)이 있나요,
아니면 수치·사례·과정 중심으로 보여줄 건가요?
```

**③ 대표 결과물 또는 성과**
```
가장 임팩트 있게 보여줄 수 있는 것 3~5가지를 알려주세요.
(수강생 수, 강의 이력, 결과물, 수치, 고객 후기 등)
```

**④ 경력·배경 포함 여부**
```
이력(경력·학력·자격)을 함께 보여줄 건가요,
아니면 결과물과 성과 중심으로만 구성할 건가요?
```

**⑤ 톤**
```
전문적·단정 / 감성·스토리텔링 / 결과 중심·데이터 강조 / 혼합
```

### 슬라이드 흐름 확정

위 답변을 바탕으로 슬라이드 흐름 초안을 제안하고 사용자와 확정한다.

흐름 예시 (목적과 성격에 따라 달라진다):

```
[결과물 중심 흐름]
커버(나는 누구) → 대표 결과물 1 → 대표 결과물 2 → 대표 결과물 3 → 후기·평가 → 연락 CTA

[경력+결과물 혼합 흐름]
커버 → 자기소개·배경 → 경력/프로젝트 1 → 경력/프로젝트 2 → 전문 분야 → 연락 CTA

[성과 수치 중심 흐름]
커버 → 핵심 수치·성과 → 주요 프로젝트 → 고객 후기 → 다음 협업 제안 → CTA
```

흐름이 확정되면 2단계로 넘어간다.

---

## 2단계 — 템플릿 선정

### 분석 기준

확정된 슬라이드 흐름과 톤을 기준으로 `brizy-story-templates.md`에서 적합한 템플릿을 분석한다.

| 기준 | 확인할 것 |
|---|---|
| 결과물 전시 구조 | 프로젝트·작업물을 슬라이드별로 보여줄 수 있는가 |
| 인물 소개 슬라이드 | 전문가 첫인상을 줄 수 있는 커버 슬라이드가 있는가 |
| 이미지 의존도 | 결과물 이미지가 많을 때와 적을 때 각각 맞는 구조인가 |
| 신뢰 증거 슬라이드 | 후기, 수치, 경력을 보여주는 슬라이드가 있는가 |
| CTA | 연락·협업 제안으로 연결되는 마지막 슬라이드가 있는가 |

### 참고 템플릿 풀

포트폴리오에 적합한 것으로 분석된 템플릿들. 이 중에서 선정하되, 다른 템플릿도 필요하면 분석한다.

| 템플릿 | 특징 | 적합한 상황 |
|---|---|---|
| CV | 커버→자기소개→경력3→전문분야→학력→CTA (8장) | 경력·이력 중심 포트폴리오 |
| Photographer | 전문가커버→작업물4→마무리 | 이미지 결과물 전시 중심 |
| Crafts | 커버→작업과정→명언→작품→갤러리 | 창작물·디자인 작업물 전시 |
| Freelancer | 포지셔닝→서비스→후기→CTA | 결과물보다 역량·서비스 소개 중심 |
| Real Estate | 명언→전문가사진→특장점→연락처 | 신뢰·전문성 강조 포트폴리오 |
| Books | 커버→스펙수치→권위→저자소개→맛보기→CTA | 콘텐츠·저작물 중심 |
| Coaching | 인물→프레임워크→주제→가치→다음단계 | 방법론·철학이 있는 전문가 |

### 1개 vs 2개 조합 결정

| 방식 | 언제 적합한가 |
|---|---|
| **1개 템플릿** | 결과물 중심 또는 경력 중심 하나로 충분할 때 |
| **2개 조합** | 자기소개(나는 누구)와 결과물 전시를 다른 톤으로 나누고 싶을 때 |
| **브릿지 슬라이드** | 2개 조합일 때, "나를 소개합니다" → "제가 만든 것들입니다" 전환 시 |

### 후보 제안 방식

후보 2~3개를 아래 형식으로 제시한다.

```
[후보 A] 템플릿명 (슬라이드 수)
구조: Slide 1 → Slide 2 → ...
이 조합을 추천하는 이유: ...
조정이 필요한 부분: ...

[후보 B] ...
```

사용자가 확정하면 3단계로 넘어간다.

---

## 3단계 — 슬라이드별 가이드 생성

확정된 템플릿 구조와 슬라이드 흐름을 기반으로
포트폴리오 정보와 셀프코어 내용을 채운 슬라이드별 표를 생성한다.

### 출력 형식

```
### Slide N — [슬라이드 제목] `[참조 템플릿 슬라이드]`

**역할:** [이 슬라이드가 포트폴리오에서 하는 역할]

| 요소 | 내용 |
|---|---|
| [요소명] | [포트폴리오 정보 또는 셀프코어에서 가져온 내용] |
| [요소명] | [포트폴리오 정보 또는 셀프코어에서 가져온 내용] |

> **팁:** [이 슬라이드 수정 시 참고할 것]
```

### 결과물 슬라이드 출력 시 주의사항

- 이미지 결과물이 있으면 슬라이드 한 장에 하나씩, 제목 + 한 줄 설명을 붙인다
- 이미지 결과물이 없을 때는 수치·성과·과정을 텍스트로 구성하는 방식을 제안한다
- 결과물이 많을 때는 "대표 3개 상세 + 나머지 목록 1장" 구조를 권장한다

### 경력 슬라이드 출력 시 주의사항

- 경력은 기관명 + 기간 + 역할 한 줄 형태로 정리한다
- 1인기업의 경우 "주요 프로젝트"나 "주요 강의 이력"으로 대체한다

---

## Claude가 하지 말아야 할 것

- 보여줄 결과물과 포트폴리오 목적 확인 없이 슬라이드를 제안하지 않는다
- 이미지가 없는 상황에서 이미지 의존도 높은 템플릿(Photographer, Crafts)만 제안하지 않는다
- CV를 기본값으로 고정해서 제안하지 않는다
- 슬라이드 흐름 확정 전에 템플릿을 먼저 선정하지 않는다
- 후보를 1개만 제안하지 않는다. 최소 2개 이상을 비교할 수 있게 제시한다
