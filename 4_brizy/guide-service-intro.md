# Brizy Story — 서비스 소개 제작 가이드

> selfcore-frame / 4_brizy / guide-service-intro.md
> 서비스 소개 스토리 제작 시 Claude가 참조하는 절차 안내

---

## 이 파일의 역할

사용자가 서비스 소개 스토리 제작을 요청할 때 Claude가 따르는 3단계 절차를 정의한다.
템플릿은 고정하지 않는다. 서비스의 성격과 전달 목적에 맞게 매번 새로 선정한다.

---

## 전체 흐름

```
1단계  콘텐츠 설계   셀프코어 확인 → 서비스 정보 협의 → 슬라이드 흐름 확정
         ↓
2단계  템플릿 선정   구성에 맞는 템플릿 분석 → 후보 제안 → 사용자 확정
         ↓
3단계  가이드 생성   확정된 템플릿 구조로 슬라이드별 텍스트 표 출력
```

---

## 1단계 — 콘텐츠 설계

### 셀프코어 확인

Project 지식에서 셀프코어와 웹사이트 표현 세트를 먼저 찾는다.

- **있으면** → 내용을 요약해서 보여주고 확인 후 다음으로 넘어간다
- **없으면** → 아래 핵심 질문으로 먼저 뽑는다

```
셀프코어 없을 때 핵심 질문

1. 어떤 분들에게 어떤 서비스를 제공하나요? (대상 + 서비스)
2. 이 서비스를 통해 고객에게 어떤 변화가 생기나요? (결과)
3. 비슷한 서비스와 무엇이 다른가요? (차별점)
```

### 서비스 정보 협의

셀프코어 확인 후 아래 항목을 하나씩 확인한다.

**① 소개할 서비스 범위**
```
하나의 대표 서비스를 집중적으로 소개할 건가요,
아니면 여러 서비스를 함께 보여줄 건가요?
```

**② 이 스토리를 보는 사람**
```
처음 알게 된 잠재 고객 / 이미 관심 있는 고객 / 소개받은 분
어느 단계의 사람에게 주로 보여주실 건가요?
```

**③ 강조하고 싶은 것**
```
서비스 내용 / 차별화 방식 / 구체적 성과·결과 / 고객 후기
중에서 가장 앞세우고 싶은 것은 무엇인가요?
```

**④ 연락·신청 방식**
```
스토리 마지막에 어디로 연결되길 원하나요?
(웹사이트 / 카카오톡 / 오픈채팅 / 신청 폼 / 전화)
```

**⑤ 톤**
```
전문적·신뢰감 / 따뜻하고 친근 / 결과 중심·임팩트 / 혼합
```

### 슬라이드 흐름 확정

위 답변을 바탕으로 슬라이드 흐름 초안을 제안하고 사용자와 확정한다.

흐름 예시 (목적과 강조점에 따라 달라진다):

```
[단일 서비스 집중 소개 흐름]
커버 → 이 서비스가 필요한 상황 → 서비스 내용 → 진행 방식 → 결과·성과 → 후기 → 연락 CTA

[다수 서비스 나열 흐름]
커버 → 서비스 1 → 서비스 2 → 서비스 3 → 차별화 방식 → 연락 CTA

[문제-해결 흐름]
이런 상황이신가요? → 원인 → 우리의 접근 → 서비스 소개 → 결과 → CTA
```

흐름이 확정되면 2단계로 넘어간다.

---

## 2단계 — 템플릿 선정

### 분석 기준

확정된 슬라이드 흐름과 톤을 기준으로 `brizy-story-templates.md`에서 적합한 템플릿을 분석한다.

| 기준 | 확인할 것 |
|---|---|
| 서비스 항목 구조 | 서비스 영역을 슬라이드별로 나눠 보여줄 수 있는가 |
| 차별화 슬라이드 | 신뢰 증거나 차별점을 보여주는 슬라이드가 있는가 |
| 연락처 CTA | 마지막에 연락처·신청 유도 슬라이드가 있는가 |
| 이미지 의존도 | 이미지 없이도 텍스트로 완성될 수 있는가 |
| 톤 | 사용자가 원하는 분위기와 맞는가 |

### 참고 템플릿 풀

서비스 소개에 적합한 것으로 분석된 템플릿들. 이 중에서 선정하되, 다른 템플릿도 필요하면 분석한다.

| 템플릿 | 특징 | 적합한 상황 |
|---|---|---|
| Freelancer | 포지셔닝→서비스항목→후기→CTA 완결 구조, 이미지 의존 낮음 | 1인기업 서비스 전반 |
| Design | 서비스영역3가지→후기→CTA | 분야별 서비스 나열 |
| App | 기능3가지→후기→CTA | 핵심 기능·서비스 3가지 소개 |
| Handymen | 카테고리목록→후기→차별점→CTA (5장) | 간결한 서비스 목록 소개 |
| Lawyer Up | 전문직 톤, 전문분야→차별점→신뢰마무리 | 고급·전문직 서비스 |
| Coaching | 인물+비주얼→프레임워크→가치→다음단계 | 코칭·컨설팅 서비스 |
| Onboarding | 핵심가치 3가지 (3장, 초간결) | 서비스 도입부 또는 단독 간결 소개 |
| Marketing | 전략→인사이트→오퍼→성과 | 마케팅·콘텐츠 관련 서비스 |

### 1개 vs 2개 조합 결정

| 방식 | 언제 적합한가 |
|---|---|
| **1개 템플릿** | 하나의 서비스 또는 명확한 흐름으로 커버될 때 |
| **2개 조합** | 서비스 소개 파트와 신뢰 구축 파트를 분리하고 싶을 때 |
| **브릿지 슬라이드** | 2개 조합일 때 두 파트 사이에 전환 문장이 필요할 때 |

### 후보 제안 방식

후보 2~3개를 아래 형식으로 제시한다.

```
[후보 A] 템플릿명 (슬라이드 수)
구조: Slide 1 → Slide 2 → ...
이 조합을 추천하는 이유: ...
조정이 필요한 부분: ...

[후보 B] ...
```

사용자가 확정하면 3단계로 넘어간다.

---

## 3단계 — 슬라이드별 가이드 생성

확정된 템플릿 구조와 슬라이드 흐름을 기반으로
서비스 정보와 셀프코어 내용을 채운 슬라이드별 표를 생성한다.

### 출력 형식

```
### Slide N — [슬라이드 제목] `[참조 템플릿 슬라이드]`

**역할:** [이 슬라이드가 서비스 소개에서 하는 역할]

| 요소 | 내용 |
|---|---|
| [요소명] | [서비스 정보 또는 셀프코어에서 가져온 내용] |
| [요소명] | [서비스 정보 또는 셀프코어에서 가져온 내용] |

> **팁:** [이 슬라이드 수정 시 참고할 것]
```

### 서비스 항목 슬라이드 출력 시 주의사항

- 서비스가 여러 개일 때 슬라이드 한 장에 하나씩 배치한다
- 각 서비스 슬라이드에는 서비스명 + 한 줄 설명 + 대상이 들어가야 한다
- 서비스가 5개 이상이면 "묶음형"(2~3개를 한 장에)도 제안한다

---

## Claude가 하지 말아야 할 것

- 서비스 범위와 연락 방식 확인 없이 슬라이드를 제안하지 않는다
- Freelancer나 Design을 기본값으로 고정해서 제안하지 않는다
- 슬라이드 흐름 확정 전에 템플릿을 먼저 선정하지 않는다
- 후보를 1개만 제안하지 않는다. 최소 2개 이상을 비교할 수 있게 제시한다
